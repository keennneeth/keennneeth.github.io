Human Computer Interaction

Project 1: Elevator Project
Kenneth Rodriguez

Find the issues with the current design.

There are a couple of bad options used in this elevator.
·One being the rear doors, to keep it simple it should just open one way, or have both doors open at the same time.
·Having double of everything makes it very cluttered, double numbers and double door open/close.
Overall it is not the worst design invented, after looking at it for a couple seconds you can understand it.

Think about the common things that you use an elevator. List your most common uses and other more rare uses. Does the interface make doing those common things easier?

People will typically use elevators in order to avoid stairs in very tall buildings, some rare times i'd use an elevator is maybe moving large objects, this interface could be practical when going up a tall building, but once you are moving items up using the elevator you could accidently click rear doors instead of front, it will be annoying.

Feedback & Common Mistakes

Many elevators will have the same functions, click a floor and it will do the rest. This elevator and most of them will highlight the floor number button red and it will let the user know it worked.
One very common mistake that people will make with this design is going to the rear doors instead of the front door or clicking the open door button and opening the wrong side.

Solution and New Design


ADD SKETCH


Simple yet effective, the new design will include a screen that will display floor level at the top of the door. The control panel will consist of 5 buttons and a screen that will display destination level. User will use up and down button to choose floor level and hit green button to go to desired floor.

Design

The reason for this is because it will allow for a clean interface and it will be easy to comprehend, using the up and down arrows will allow the user to pick a desired floor, after a couple seconds of being on that number the door will automatically close or they can click the green button to speed up the process. This will eliminate the close door button, depending on how tall the building is, it will eliminate a set amount of buttons.

Design Options

This can be made with a touch screen if desired, or just simple buttons, both will have the same functions but one will look modern. The screen will be about 6" tall and 10" width, The reason for this is because the screen doesnt have to be large in order to fit 2 buttons and a number. The feedback for this will be a speaker announcing "Going Up", or "Going down", depending on what the user selects.
An information screen can be brought up by clicking a series of buttons that will display the maintence info and anything important that can be needed.

Disabilities

People with disabilities will be able to use the elevator, making it bigger than a normal elevator you can comfortably fit someone on a wheel chair. Also by adding brail to the buttons on the non touch screen, it will allow blind people to use it.
